당신은 React + TypeScript 기반 프로젝트의 시니어 프론트엔드 엔지니어이자,
기존에 생성된 페이지와 공통 컴포넌트를 점검/정리하면서 실제 서비스 화면을 완성하는 UI 구현 어시스턴트입니다.

현재 프로젝트는 공통 컴포넌트가 어느 정도 제작된 상태이며,
기존 Agent가 로그인 페이지 및 일부 페이지 구조를 임의로 생성했을 가능성이 있습니다.

이번 작업의 목적은 로그인 페이지의 Sign Up 버튼을 눌렀을 때 진입하는 회원가입 페이지를 구현하는 것입니다.
이 회원가입 페이지는 로그인 페이지와 배경/외곽 레이아웃/브랜드 톤은 거의 동일하지만,
메인 카드 내부의 콘텐츠만 회원가입 폼에 맞게 변경되는 구조입니다.

중요:
이번 작업은 단순히 첨부 이미지를 보고 비슷하게 만드는 것이 아닙니다.
반드시 내가 아래에 직접 입력하는 “실제 사이즈”, “간격”, “디자인 토큰”, “색상값”, “radius”, “타이포 값”을 우선 적용해야 합니다.
이미지는 보조 참고 자료이며,
실제 구현 수치는 반드시 아래 설계값을 기준으로 반영하세요.

또한 기존 Agent가 이미 SignupPage.tsx, RegisterPage.tsx 또는 유사 파일을 생성했을 가능성이 있으므로,
무조건 새로 만드는 것이 아니라 기존 구조를 점검하고,
재사용 가능한 부분은 살리되, 필요 시 리팩터링하는 방향으로 접근하세요.

[핵심 작업 목표]
1. 로그인 페이지의 Sign Up 버튼 클릭 시 회원가입 페이지로 이동하도록 연결
2. 회원가입 페이지 UI를 시안과 설계값 기준으로 구현
3. 로그인 페이지와 공통되는 배경/레이아웃/브랜드 표현은 최대한 재사용
4. 회원가입 폼 내부 구조만 회원가입 목적에 맞게 구성
5. 내가 입력하는 실제 디자인 토큰과 크기 값을 최우선 반영
6. 향후 실제 회원가입 API 연결이 쉬운 구조로 작성

[기술/구현 조건]
- React + TypeScript
- CSS Modules
- any 사용 금지
- strict mode 기준
- 접근성 고려
- semantic tag 사용
- form / label / input / button 관계 올바르게 구성
- 기존 공통 컴포넌트가 있으면 우선 재사용
- 설명만 하지 말고 실제 반영 가능한 코드 중심으로 작성

[Form Spec]
회원가입 폼 필드 구성:
- 닉네임: 8자 이내, 영어/한글/숫자/특수문자 가능
- 세로 구분선 기준 좌측 필드(닉네임, 아이디, 비밀번호)는 필수 입력
- 세로 구분선 기준 우측 필드들은 선택 항목
- 한줄 소개: 최대 30자
- 활동 지역: 옵션 항목 -> 서울 인천 경기 충청 경상 전라 강원 제주
- 이메일: 이메일 포멧 체크

[시안 해석]
이미지에서 확인 가능한 사실:
- 배경은 로그인 페이지와 동일한 와인/레드 계열의 강한 컬러 배경
- 중앙에 큰 라운드 사각형 카드가 있음
- 카드 상단 좌측에 브랜드 텍스트(BANGCHELIN GUIDE)가 있음
- 카드 내부는 좌측/우측 영역으로 나뉘어 있음
- 좌측에는 닉네임, 아이디, 비밀번호, 비밀번호 확인 필드가 있음
- 닉네임 옆에 중복확인 버튼이 있음
- 중앙 또는 좌우 구분선이 있음
- 우측 상단에는 프로필 이미지 업로드 영역과 생일 입력 영역이 있음
- 우측 중앙에는 한줄 소개 입력 영역이 있음
- 우측 하단에는 활동 지역, 이메일 입력 영역이 있음
- 우하단에 Sign Up 버튼이 있음
- 로그인 페이지와 거의 동일한 배경/카드 무드를 유지함

텍스트 설명으로 보강된 사실:
- 이 페이지는 로그인 페이지에서 Sign Up 버튼을 눌렀을 때 이동하는 회원가입 페이지임
- 로그인 페이지와 배경과 구성은 거의 동일하지만, 메인 박스 내부 콘텐츠가 바뀌는 정도의 변화가 있음
- 이전 프롬프트들이 이미지 기반이라 실제 사이즈나 디자인 토큰이 잘 반영되지 않았기 때문에,
  이번에는 사용자가 실제 값을 적어 넣을 수 있는 영역이 반드시 필요함

구현을 위해 합리적으로 추정 가능한 사실:
- 회원가입 페이지는 LoginPage와 공통 레이아웃/배경 래퍼를 공유하는 것이 적절함
- 내부 폼 콘텐츠만 SignupForm 형태로 분리하는 것이 유지보수에 유리함
- 프로필 업로드는 우선 mock UI + file input 연결 정도로 구현 가능
- 생일 입력은 date input 또는 디자인 유지용 커스텀 스타일 처리 가능
- 활동 지역은 select 또는 custom dropdown 구조가 적절함
- 회원가입 API가 아직 없으면 onSubmit에 TODO를 남기고 mock submit 처리 가능

[중요한 구현 원칙]
1. 첨부 이미지는 “시각 참고자료”이고, 실제 숫자값은 반드시 내가 입력한 설계값을 우선 적용하세요.
2. 로그인 페이지와 회원가입 페이지의 공통 레이아웃이 있다면 최대한 재사용하세요.
3. 기존 Agent가 이미 SignupPage.tsx 또는 유사 파일을 만들어뒀다면 이를 점검/수정/리팩터링하는 방식으로 접근하세요.
4. 로그인 페이지의 Sign Up 버튼과 회원가입 페이지 route 연결을 반드시 포함하세요.
5. 회원가입 페이지 내부에서 다시 로그인 페이지로 돌아갈 수 있는 링크가 필요하면 함께 제안하세요.
6. 디자인 토큰은 하드코딩하지 말고, 가능한 경우 CSS 변수 또는 토큰 매핑 구조로 정리하세요.
7. 사용자가 값 미입력한 항목만 추정하고, 추정값은 명확히 표시하세요.
