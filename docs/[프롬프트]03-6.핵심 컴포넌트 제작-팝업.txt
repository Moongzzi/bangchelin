[레이아웃/구조 해석]
이미지에서 명확히 확인 가능한 사실:
- 팝업은 중앙 정렬되어 있음
- 패널은 밝은 배경의 카드 형태임
- 패널에는 제목, 설명, 버튼 영역이 있음
- 버튼 조합은 한 개 또는 두 개일 수 있음
- 동일 레이아웃에 버튼 스타일만 달라지는 variation이 존재함
- 패널은 고정 폭처럼 보이지만, 실제 서비스 요구사항상 콘텐츠에 따라 유동 크기여야 함

텍스트 설명으로 보강된 사실:
- overlay는 text/primary-40% 계열 컬러 박스로 사이트 전체를 덮음
- overlay가 있는 동안 사이트 뒤쪽은 클릭/스크롤이 되지 않아야 함
- 모바일 반응형도 고려해야 함

구현을 위해 합리적으로 추정하는 사실:
- Popup은 Portal 기반 렌더링이 적절할 가능성이 높음
- body scroll lock이 필요함
- overlay 클릭 닫기 여부는 옵션화 가능함
- ESC 닫기 여부도 옵션화 가능함
- 패널의 width는 콘텐츠 기반이되, min/max width가 있어야 함
- 긴 콘텐츠는 내부 scroll 또는 body 영역 scroll을 지원하는 것이 적절함

[중요한 overlay 처리 규칙]
- 팝업이 열리면 사이트 전체를 덮는 overlay를 반드시 렌더링해야 함
- overlay는 text/primary-40% 계열 컬러를 우선 사용해야 함
- 현재 exact token 값이 명시되어 있지 않으면:
  1) 기존 토큰 참조 위치를 먼저 찾을 것
  2) 없다면 popup overlay 전용 token 또는 CSS variable로 분리할 것
- overlay가 떠 있는 동안:
  - 배경 클릭 차단
  - 배경 스크롤 차단
  - 배경 포커스 이동 차단 또는 최소화
- 구현 시 overlay 색상을 임의의 검정 rgba로 바로 하드코딩하지 말고,
  가능하면 토큰 또는 CSS variable로 먼저 분리하세요.

[레이아웃/상태 규칙]
동작:
- open / close 제어 가능해야 함
- onClose 핸들러 제공
- 필요 시 overlay click 닫기 옵션 지원
- 필요 시 ESC 닫기 옵션 지원
- 버튼 클릭 시 각 action handler 실행 가능해야 함

정렬 규칙:
- title, description은 중앙 정렬 우선
- actions는 하단 영역에서 시안에 맞는 정렬 방식 유지
- 버튼 2개일 때 간격 유지
- 버튼 1개일 때 자연스럽게 단독 배치

크기 규칙:
- popup panel은 내부 콘텐츠에 따라 유동적 크기
- 단, max-width / max-height 필요
- 모바일에서는 viewport를 넘지 않도록 width 축소
- 필요 시 내부 콘텐츠 영역 scroll 허용

상태 처리 방향:
- open / closed
- action variant
- single / double action
- dismissible / non-dismissible
- loading action은 아직 과도하게 구현하지 말고 확장 가능성만 고려

[디자인 토큰/스타일 참고]
확정된 컬러 토큰:
- text/primary : text
- bg/elevated : background
- border/strong : Outline
- brand/primary : primary button background, Outline button outline&text 
- text/inverse : primary button text

추가 스타일 해석:
- panel 배경은 밝은 neutral/surface 계열
- title과 description은 중앙 정렬된 텍스트 구조
- 버튼은 기존 공통 Button variant를 재사용하는 것이 적절함
- panel radius는 비교적 둥근 편
- overlay는 text/primary-40% 계열로 분리하는 것이 적절함
- 그림자/레이어가 필요하면 foundation 수준으로 최소 정의
