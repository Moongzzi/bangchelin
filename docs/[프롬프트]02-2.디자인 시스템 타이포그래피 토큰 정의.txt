당신은 React + TypeScript 기반 프론트엔드 프로젝트의 디자인 시스템 설계를 돕는 AI 어시스턴트입니다.
지금까지 작업한 내용은 [프롬프트]로 시작되는 텍스트 파일들로 넘버링해서 관리 중입니다.
이전 요청 사항들을 먼저 확인해서 현재 프로젝트를 파악한 뒤 아래 작업을 진행해주세요.

현재 작업 단계는 Step 2-2이며, 목적은 디자인 시스템의 타이포그래피 토큰을 정의하는 것입니다.
이번 단계에서는 페이지 구현이나 컴포넌트 구현까지 진행하지 말고, 타이포그래피 토큰 설계에만 집중해주세요.

## 프로젝트 전제
- 웹 서비스 프로젝트입니다.
- React + TypeScript strict 모드를 사용합니다.
- 재사용성과 유지보수성을 우선합니다.
- 하드코딩된 폰트 스타일 사용을 최소화하고, 토큰 기반 스타일 구조를 우선합니다.
- 기존 공통 컴포넌트(Button, InputField, Dropdown, Toggle, RadioButton, Checkbox, Popup, SearchBar, ListView, Header, Footer)에 추후 쉽게 적용할 수 있어야 합니다.
- 접근성과 가독성을 기본적으로 고려해야 합니다.
- 정보량이 많은 UI에서도 시각적 위계가 명확해야 합니다.

## 디자인 방향
- 전체 분위기는 고급스럽고 정제된 느낌을 지향합니다.
- 과도하게 화려하거나 실험적인 타이포그래피보다는 절제된 프리미엄 무드를 원합니다.
- 미슐랭 가이드와 유사한 신뢰감 있는 정보형 UI를 참고합니다.
- 필요 시 약간의 고딕 무드를 섞을 수 있지만, 가독성과 안정감을 해치면 안 됩니다.
- 명확한 정보 전달이 가장 중요합니다.

## 가장 중요한 규칙
타이포그래피는 semantic token 중심으로 설계해주세요.
단순히 font-size 목록만 나열하지 말고, 실제 UI에서 재사용 가능한 역할 기반 토큰 체계로 정리해주세요.
예: display, heading, title, body, label, caption 등

## 폰트 패밀리 고정 조건
프로젝트에서 사용할 폰트 방향은 아래와 같습니다.

- 영어(라틴 문자) 중심 표현: GmarketSans
- 한글 중심 표현: Pretendard

위 조건을 반드시 반영해서 fontFamily 토큰을 설계해주세요.

단, 실제 CSS에서는 언어별 자동 분기와 fallback 전략이 필요할 수 있으므로,
다음 관점까지 함께 고려해서 제안해주세요.

- latin 계열 텍스트용 fontFamily
- korean 계열 텍스트용 fontFamily
- 기본 UI용 공통 fontFamily
- heading/title 계열에서 어떤 폰트를 우선 사용할지에 대한 제안

필요하다면 `latin`, `korean`, `base`, `heading`처럼 역할 기반으로 fontFamily 토큰을 나눠도 됩니다.

## 값 정의 방식
- fontFamily는 위 조건을 기준으로 명확히 정의해주세요.
- fontWeight, fontSize, lineHeight, letterSpacing의 구체적인 값은
  프로젝트 성격(고급스럽고 정제된 정보형 UI)에 맞게 합리적으로 제안해주세요.
- 지나치게 공격적이거나 실험적인 수치 대신,
  실제 서비스에서 바로 적용 가능한 안정적인 기본값을 우선 제안해주세요.

## 타이포그래피 설계 목표
- light theme 기준 UI에 잘 어울리는 안정적인 타이포그래피 체계를 제안할 것
- 정보형 서비스에 적합한 읽기 쉬운 본문 중심 구조를 만들 것
- 헤드라인보다 body, label, UI text의 완성도를 우선할 것
- 추후 dark theme에서도 무리 없이 재사용 가능한 구조를 고려할 것
- 데스크톱 우선이지만, 반응형 확장이 가능하도록 설계할 것
- 한국어 UI 가독성을 충분히 고려할 것
- 숫자, 리스트, 메타 정보, 버튼 텍스트 등 다양한 정보성 요소에 잘 맞아야 할 것

## 반드시 포함할 타이포그래피 카테고리
아래 범주를 포함해서 설계해주세요.

- fontFamily
- fontWeight
- fontSize
- lineHeight
- letterSpacing
- semantic typography tokens

semantic typography tokens에는 아래와 같은 역할 기반 항목을 포함해주세요.
- display
- heading
- title
- body
- label
- caption

필요하다면 각 카테고리를 2~4단계 정도로 세분화해주세요.
예:
- heading.xl / heading.lg / heading.md
- body.lg / body.md / body.sm
- label.md / label.sm
- caption.md / caption.sm

## 토큰 설계 조건
- semantic token 중심으로 설명할 것
- primitive token과 semantic token의 관계를 구분해서 제안할 것
- fontFamily, fontWeight, fontSize, lineHeight, letterSpacing은 primitive 또는 foundation token으로 제안할 것
- display, heading, title, body, label, caption은 semantic typography token으로 제안할 것
- 토큰 이름은 직관적이고 확장 가능해야 할 것
- React + TypeScript에서 바로 참조 가능한 구조여야 할 것
- 지나치게 많은 단계보다, 적고 안정적인 단계 구성을 우선할 것
- 정보형 UI에서 반복적으로 쓰기 좋은 구조여야 할 것
- 버튼, 입력창, 리스트, 팝업, 헤더, 푸터 등에 공통 적용하기 쉬워야 할 것
