[레이아웃/구조 해석]
이미지에서 명확히 확인 가능한 사실:
- 전체적으로 가로로 긴 직사각형 input box 형태임
- 입력 영역 왼쪽에 텍스트가 위치함
- 오른쪽에 검색 아이콘이 있음
- placeholder처럼 보이는 연한 텍스트 상태가 있음
- 실제 입력된 값처럼 보이는 더 진한 텍스트 상태가 있음
- 마지막 예시에서는 커서가 보이는 상태로 해석 가능함
- 전체적으로 밝은 톤의 필드와 얇은 보더가 보임

텍스트 설명으로 보강된 사실:
- 이 컴포넌트는 search box / search input 계열 컴포넌트임
- 어둡게 보이는 배경이 있다면 실제 다크 배경색이 아니라 transparent 때문에 뒤 배경이 비친 것일 수 있음

구현을 위해 합리적으로 추정하는 사실:
- input type="search" 또는 text 기반 검색 입력 구조가 적절함
- 우측 검색 아이콘은 장식용이거나 submit action 트리거로 확장 가능함
- 현재 단계에서는 기본 입력 컴포넌트 + 아이콘 구조가 적절함
- clear button, 자동완성, 추천검색어는 이번 단계에서 제외하는 것이 적절함

중요한 배경색 해석 규칙:
- 서치 박스의 배경이 어둡게 보이더라도 이를 실제 고정 dark background color로 단정하지 마세요.
- 투명 배경(transparent) 때문에 부모 또는 뒤 배경색이 비쳐 어둡게 보일 가능성을 우선 고려하세요.
- 따라서 background는 아래 우선순위로 판단하세요:
  1) 디자인 토큰 또는 명시된 컬러 정보가 있으면 그것을 사용
  2) 명시 정보가 없고 어둡게 보이는 이유가 투명 배경일 가능성이 크면 transparent 가능성을 열어둘 것
  3) 단정이 어려우면 “배경은 transparent 또는 밝은 overlay 계열로 추정”이라고 명시할 것
- 구현 시에는 배경색을 검정/짙은 회색으로 하드코딩하지 마세요.
- 필요하면 background를 configurable prop 또는 token으로 분리하세요.

[레이아웃/상태 규칙]
동작:
- 사용자는 텍스트를 입력할 수 있어야 함
- controlled / uncontrolled 모두 대응 가능한 구조면 좋음
- Enter 입력 시 onSubmit 또는 onSearch 핸들러 확장 가능성 고려
- 아이콘 클릭 시 검색 액션을 연결할 수 있는 구조면 좋지만, 기본 구현은 과도하게 복잡하지 않게 유지

정렬 규칙:
- 입력 텍스트는 좌측 정렬
- 검색 아이콘은 우측 정렬
- 텍스트와 아이콘 사이에 충분한 여백이 있어야 함
- 아이콘이 input 텍스트 영역을 침범하지 않도록 구조화할 것

상태 처리 방향:
- placeholder / filled / focused 상태를 표현할 수 있어야 함
- disabled는 HTML 속성과 시각 스타일이 함께 반영되도록 설계
- readOnly를 선택적으로 지원 가능하게 설계
