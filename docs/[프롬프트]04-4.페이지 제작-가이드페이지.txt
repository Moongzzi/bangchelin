당신은 React + TypeScript 기반 프로젝트의 시니어 프론트엔드 엔지니어입니다.
기존에 공통 레이아웃, 헤더, 푸터, 일부 페이지 및 스타일이 이미 생성되어 있을 수 있으며,
이전 Agent가 화면 이미지를 바탕으로 UI를 임의 구현했을 가능성도 있습니다.

이번 작업의 목표는 가이드 페이지 AboutPage.tsx를 구현하는 것입니다.

이 페이지는 다음과 같은 요구사항을 가집니다.

[핵심 요구사항]
1. 페이지 좌측에는 트리뷰(Tree View) 구조의 사이드 메뉴가 있어야 합니다.
2. 트리뷰는 최대 3뎁스까지만 지원합니다.
3. 우측 콘텐츠 영역 상단에는 "편집" 버튼이 위치할 수 있습니다.
4. 이 편집 버튼은 관리자 권한이 있는 경우에만 보여야 합니다.
5. 관리자 권한 여부는 추후 API로 확인할 예정이므로,
   현재는 mock 상태값 / placeholder 훅 / 임시 boolean 변수 등으로 구조만 먼저 설계하세요.
6. 이 페이지는 콘텐츠 영역에서 좌우 여백 없이 창 전체 폭을 활용하도록 설계되어 있습니다.
7. 첨부된 시안 이미지는 참고용이며,
   실제 구현 시에는 내가 아래에 작성할 실측 사이즈, 간격, 토큰 값을 최우선 적용해야 합니다.

중요:
이번 작업은 단순히 이미지를 보고 비슷하게 만드는 것이 아니라,
“실제 설계값과 디자인 토큰을 우선 적용할 수 있는 구조로 페이지를 구현하는 것”이 목적입니다.
이미지는 보조 참고자료일 뿐이며,
내가 아래에 입력하는 값이 있으면 반드시 그 값을 우선 적용하세요.
값이 비어 있는 경우에만 합리적으로 추정하고, 추정값은 명시하세요.

[시안 해석]
이미지에서 확인 가능한 구조:
- 상단에는 공통 헤더가 있음
- 좌측에 세로형 사이드 메뉴 영역이 있음
- 사이드 메뉴는 아코디언/트리뷰처럼 접고 펼칠 수 있는 구조로 보임
- 일부 메뉴는 active 상태가 존재함
- 우측은 넓은 콘텐츠 영역이며, 좌우 여백을 크게 두지 않고 전체 폭을 넓게 사용함
- 우측 상단에 편집 버튼이 배치되어 있음
- 본문은 제목, 섹션 제목, 설명 텍스트 등 문서형 콘텐츠 구조임
- 좌측 메뉴와 우측 콘텐츠는 명확히 분리되어 있음
- 하단에는 공통 푸터가 있음

텍스트 요구사항으로 확정된 내용:
- 왼쪽에 트리뷰 구조의 사이드 메뉴 제공
- 최대 3뎁스까지만 제공
- 1,2뎁스는 선택 시, 콘텐츠 내용이 변경되는 것이 아닌 사이드 메뉴 드롭다운만 열리고 닫히는 형태.
- 3뎁스 선택 시에 콘텐츠 내용이 변경되며, 1 뎁스를 기준으로 모두 한 페이지에 작성되며, 그 중 2뎁스-3뎁스의 위치에 해당하는 위치로 자동 스크롤 되어야 함
- 관리자 권한이 있는 경우에만 우측의 편집 버튼 표시
- 관리자 권한은 추후 API로 확인 예정
- 콘텐츠 영역은 좌우 여백 없이 창 전체를 활용

구현을 위해 합리적으로 추정 가능한 사항:
- AboutPage는 문서/가이드/정책성 콘텐츠를 표시하는 페이지일 가능성이 높음
- 좌측 메뉴는 현재 선택된 문서/섹션을 표시해야 함
- 3depth 구조는 재귀 컴포넌트 또는 depth 제한 로직으로 처리 가능
- 관리자 권한은 useAdminPermission 같은 훅 또는 mock boolean으로 추후 API 대체 가능하게 설계하는 것이 좋음
- 편집 버튼 클릭 시 편집 페이지 이동 또는 편집 모드 진입 구조로 확장 가능
- 콘텐츠는 현재 mock 데이터로 두고 나중에 CMS/API 데이터로 교체 가능하도록 설계하는 것이 좋음

[중요 구현 원칙]
1. 첨부 이미지는 참고용이며, 실제 수치는 내가 입력하는 설계값을 우선 적용하세요.
2. 기존 Agent가 AboutPage.tsx 또는 유사 페이지를 이미 만들었을 수 있으므로,
   무조건 새로 만들기보다 기존 구조를 점검하고 재사용 가능한 부분은 유지하세요.
3. 트리뷰는 최대 3뎁스까지만 허용해야 하며,
   4뎁스 이상 데이터가 들어오더라도 렌더링되지 않거나 예외 처리되도록 방어 로직을 두세요.
4. 관리자 권한 로직은 아직 API가 없으므로,
   추후 API 연결이 쉬운 구조(예: hook, selector, permission util)로 분리하세요.
5. 관리자 권한이 없으면 편집 버튼은 렌더링하지 마세요.
6. 콘텐츠 영역은 좌우 여백 없이 넓게 활용하되,
   필요하다면 내부 텍스트 가독성을 위한 본문 래퍼를 선택적으로 둘 수 있도록 구조를 제안하세요.
7. 스타일 값은 가능하면 CSS 변수 또는 tokens/constants로 관리하세요.
8. 하드코딩된 임의 수치 사용은 최소화하고,
   “사용자가 직접 수정할 값” 구조를 먼저 제시한 뒤 그 값을 참조하는 방식으로 작성하세요.
