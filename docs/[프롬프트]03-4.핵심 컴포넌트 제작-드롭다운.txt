당신은 React + TypeScript 기반 프로젝트의 시니어 프론트엔드 엔지니어이자 시안 기반 UI 컴포넌트 설계 어시스턴트입니다.

이번 작업은 첨부된 드롭다운(dropdown/select) 이미지 구현에만 집중해주세요.

첨부한 이미지는 Bangchelin Guide의 Dropdown 시안입니다.
라벨, 선택된 값 텍스트, 우측 chevron 아이콘, 사각형 field border, 닫힌 상태와 열린 상태의 menu panel,
옵션 리스트 등 반복되는 UI 구조를 분석해서
재사용 가능한 Dropdown 컴포넌트로 설계해주세요.

목표는 특정 화면을 하드코딩하는 것이 아니라,
이미지에 나타난 반복 구조를 바탕으로
실제 프로젝트에서 재사용 가능한 Dropdown 컴포넌트를 설계하고 구현하는 것입니다.

[레이아웃/구조 해석]
이미지에서 명확히 확인 가능한 사실:
- label이 field 위에 위치함
- field는 사각형 border를 가진 box 형태임
- field 내부 왼쪽에 텍스트가 있음
- field 내부 오른쪽에 chevron 아이콘이 있음
- 닫힌 상태에서는 아이콘이 아래 방향임
- 열린 상태에서는 아이콘이 위 방향임
- 열린 상태에서는 field 아래에 큰 직사각형 menu panel이 연결되어 있음
- menu panel 내부에 option 텍스트가 세로로 나열되어 있음
- 전체적으로 투명한 background 위에 light border 조합처럼 보임
- 동일한 닫힌 구조가 2회 반복되고, 우측에는 열린 구조 예시가 있음

텍스트 설명으로 보강된 사실:
- 이 컴포넌트는 dropdown/select 계열 컴포넌트임
- 우측 패널은 열려 있는 상태(open state) 예시로 해석 가능함
- 옵션은 단일 선택용 리스트로 해석하는 것이 자연스러움

구현을 위해 합리적으로 추정하는 사실:
- label + trigger + menu + option 구조가 적절함
- single select 기준 구현이 적절함
- trigger는 button 역할, menu는 listbox 역할, option은 option 역할을 갖는 구조가 적절함
- option 선택 시 현재 값이 trigger에 반영되고 menu는 닫히는 구조가 적절함
- 바깥 클릭 시 닫힘, Esc 키 닫힘, 방향키 이동 정도까지는 기본 인터랙션으로 고려하는 것이 적절함
- 시안상 option 좌측 아이콘, 체크마크, 섹션 구분선은 보이지 않으므로 이번 단계에서 제외하는 것이 적절함

[레이아웃/상태 규칙]
드롭다운 동작:
- 기본은 single select
- trigger 클릭 또는 키보드 조작으로 open/close 가능
- option 선택 시 현재 값 반영
- 선택 후 menu 닫힘
- 바깥 클릭 시 닫힘 처리 가능
- Esc 키로 닫힘 처리 가능
- ArrowUp / ArrowDown 으로 option 이동 가능하면 좋음
- Enter / Space로 열기 및 선택 가능하도록 고려

정렬 규칙:
- label은 trigger 위에 위치
- 값 텍스트는 trigger 내부 좌측 정렬
- chevron 아이콘은 trigger 내부 우측 정렬
- menu는 trigger 아래에 위치
- option 텍스트는 좌측 정렬

상태 처리 방향:
- closed/open 상태를 지원해야 함
- selected 값 반영이 가능해야 함
- disabled는 HTML 속성과 시각 스타일이 함께 반영되도록 설계
- 내부 state와 제어형 props(open, onOpenChange) 모두 대응 가능한 방향이면 좋음
