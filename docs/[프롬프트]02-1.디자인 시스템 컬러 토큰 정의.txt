당신은 React + TypeScript 기반 프론트엔드 프로젝트의 디자인 시스템 설계를 돕는 AI 어시스턴트입니다.

현재 작업 단계는 Step 2-1이며, 목적은 디자인 시스템의 컬러 토큰을 정의하는 것입니다.
이번 단계에서는 페이지 구현이나 컴포넌트 구현까지 진행하지 말고, 컬러 토큰 설계에만 집중해주세요.

## 프로젝트 전제
- 웹 서비스 프로젝트입니다.
- React + TypeScript strict 모드를 사용합니다.
- 재사용성과 유지보수성을 우선합니다.
- 하드코딩된 색상값 사용을 최소화하고, 토큰 기반 스타일 구조를 우선합니다.
- 기존 공통 컴포넌트(Button, InputField, Dropdown, Toggle, RadioButton, Checkbox, Popup, SearchBar, ListView, Header, Footer)에 추후 쉽게 적용할 수 있어야 합니다.
- 접근성과 가독성을 기본적으로 고려해야 합니다.
- 정보량이 많은 UI에서도 시각적 위계가 명확해야 합니다.

## 디자인 방향
- 전체 분위기는 고급스럽고 정제된 느낌을 지향합니다.
- 과도하게 화려하거나 트렌디한 색감보다는 절제된 프리미엄 무드를 원합니다.
- 미슐랭 가이드와 유사한 신뢰감 있는 정보형 UI를 참고합니다.
- 필요 시 약간의 고딕 무드를 섞을 수 있지만, 가독성과 안정감을 해치면 안 됩니다.
- 명확한 정보 전달이 가장 중요합니다.

## 이번 요청 범위
이번에는 컬러 토큰만 정의해주세요.

## 토큰 설계 조건
- semantic token 중심으로 설명하되, 실제 값은 위 CSV 기반 토큰을 그대로 반영할 것
- light theme 기준으로 먼저 설계할 것
- 이후 dark theme로 확장 가능한 구조를 고려할 것
- brand / accent / background / surface / border / text / semantic 구조를 유지할 것
- 각 토큰의 역할을 명확히 설명할 것
- primary와 accent의 역할 차이를 구분할 것
- text와 background, surface 사이의 사용 맥락을 구분할 것
- 상태 색상은 정보형 서비스에 어울리게 과하지 않게 유지할 것
- 접근성과 텍스트 대비를 고려할 것

## 구현 규칙
- 먼저 3~5줄 이내의 구현 계획을 작성해주세요.
- 그다음 제안 파일 경로를 작성해주세요.
- 이번 단계에서는 컬러 토큰 정의만 하세요.
- spacing, typography, component style까지 확장하지 마세요.
- 관련 없는 파일은 수정하지 마세요.
- 불필요한 라이브러리 추가는 금지합니다.
- any 사용은 금지합니다.
- TypeScript에서 바로 사용할 수 있는 객체 형태로 작성해주세요.
- 가능하면 as const를 사용해주세요.
- 객체 key는 JavaScript/TypeScript에서 쓰기 좋은 camelCase 또는 명확한 중첩 구조로 정리해주세요.
- 단, 값은 위 CSV 기준을 그대로 유지해주세요.

## 추가 요청
- 단순히 색상값만 나열하지 말고, 왜 이런 그룹 구조가 적절한지 짧게 설명해주세요.
- brand는 핵심 액션 컬러, accent는 보조 강조 컬러로 역할을 구분해주세요.
- background / surface / border는 정보형 UI에 맞게 절제된 계층 구조로 설명해주세요.
- text 토큰은 정보 위계 중심으로 설명해주세요.
- semantic 토큰은 상태 표현용으로만 사용하는 원칙을 설명해주세요.
- 실제 컴포넌트 적용 코드는 이번 단계에서 작성하지 마세요.
